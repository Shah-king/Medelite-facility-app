Medelite Technical Case Study: Facility Assessment Report Generator 
 
AI USE IS ENCOURAGED 
 
1. Project Objective 
As a Director at Medelite, my team routinely evaluates skilled nursing facilities before initiating outreach or partnership discussions. Currently, gathering a facility's performance metrics, star ratings, and hospitalization data requires digging through separate public databases and internal notes. 
Your task is to build a lightweight web application (a localized "micro-app" or report generator) that allows a user to input a facility's unique identifier to instantly pull public data, combine it with manual operational inputs, and download a polished summary report. 
2. Core Functional Requirements (The Required MVP) 
To pass the initial technical review, your application must successfully implement the following: 
? Dynamic CCN Lookup: An input box where the user can enter any valid CCN (CMS Certification Number) to dynamically fetch data for that specific facility. 
? The Data Engine: Use the entered CCN to query the public CMS Provider Data Catalog API and instantly fetch the facility's location, star ratings, and metadata. 
? Facility Name Override: By default, populate the official legal name from the CMS API. However, because Medelite frequently utilizes localized internal names, you must provide an optional text field allowing the user to type a custom name, which will override the API's name on the final output. 
? Manual Operational Inputs: Input fields for internal metrics that do not live in the public CMS database (EMR, Current Census, Patient Type, and Medelite History). 
? One Polished PDF Export: A single action button to "Download PDF" that instantly compiles the data and triggers a direct browser download of a clean, print-ready document. 
? Medicare Source Hyperlink: The generated PDF must include a clickable hyperlink somewhere in the layout linking back to the official Medicare Care Compare source profile using the dynamic CCN. 
? Example Target URL Format: 
[https://www.medicare.gov/care-compare/details/nursing-home/105447](https://w ww.medicare.gov/care-compare/details/nursing-home/105447) (where 105447 is dynamically replaced by the searched CCN). 
? Deployment: A live, working URL link to your hosted application alongside your public code repository. 
3. Optional Bonus Features (Completely Optional) 
If you finish the MVP early and want to showcase advanced engineering depth, consider implementing any of the following: 
? All 12 Hospitalization/ED Metrics: Pull and map the full suite of short-term (STR) and long-term (LT) hospitalization/ED metrics along with their respective state and national averages from the CMS claims-based datasets. 
? Word Doc Export (.docx): Add an additional action button to download the report as an editable Word document. 
? Complex Charts/Cards: Display the performance metrics using responsive data cards or visual charts (e.g., using Chart.js or Recharts). 
? Advanced Error Handling: Build comprehensive boundaries for edge cases (e.g., handling invalid CCNs or missing API data fields cleanly).
